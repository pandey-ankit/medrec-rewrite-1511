The gateways in MedRec are similar to the repositories, because both of them encapsulate the Java EE APIs to access external resources. One can think of repositories as a kind of gateway. The major difference between the two include:

  • Repositories encapsulate only persistence stores like relational databases, file stores, etc, while gateways encapsulate other resources, including various middleware.
  • Within MedRec, there are multiple repositories and each corresponds to an independent entity, but there is only one generic gateway for each type (such as messaging or naming). These gateways come directly from the common components; MedRec does not include any application-specific (or even use-case-specific) ones because the application currently requires only simple gateways.

In MedRec, the gateways completely encapsulate the Java EE APIs including JMS, JNDI, JavaMail, and so on. See sections on messaging, naming, mail, and so on in the Common Components chapter for the related details.
