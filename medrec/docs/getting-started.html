While installing WebLogic Server, if you chose the "Complete with Samples" option to install the Examples domain, and then used the Quick Start Configuration Wizard option to automatically install the samples templates, then the default MedRec domain (non-MT single server) has already been populated at @MW_HOME/user_projects/domains/medrec.

The following sections will help you get started using and learning about MedRec:

  • Starting and Using MedRec

  • Building the MedRec Application Suite

Starting and Using MedRec

When running the WebLogic Server installation program, select the Complete Installation option to install the Examples domain, which allows you to run the Avitek Medical Records application suite.

To begin using the MedRec application, complete these steps:

  1. Change to the @MW_HOME/user_projects/domains/medrec directory.

    On Windows:

    Enter the following command:

     startWebLogic.cmd

    Or on UNIX Bourne shell:

    Enter the following command:

     sh ./startWebLogic.sh
  2. After the server boots, a browser window opens and loads the main Avitek Medical Records index page, http://hostname:port/medrec/start.xhtml. Read this page to learn about the basic user roles supported in MedRec.
  3. To begin using the MedRec application, use a link on the index page or point a new browser window to http://hostname:port/medrec/index.xhtml. The Main MedRec application page provides log-in links for the three MedRec user types: Patients, Physicians, and Administrators.

    4. Note: You can register new patients in the system by using the Register link on the start page. Physicians, Administrators and a some Patients are pre-defined, and you can login as a certain role by using the passwords listed at http://hostname:port/medrec/index.xhtml and http://hostname:port/medrec/physician/login.xhtml.

  4. To try out the basic features of MedRec, perform the following tasks:
    1. Register a new patient using the Register link on http://hostname:port/medrec/index.xhtml.
    2. Log in as an Administrator and approve the new registration request.
    3. Log in as a Physician and create new Medical Records for the patient.
    4. Log in as a new patient and view the newly created medical records.

Invoking and Using the MedRec Administration Console

To invoke the MedRec Administration Console:

  1. Enter the following URL in your browser: http://hostname:port/console.

    where hostname refers to the computer on which MedRec is running.

    If your browser is on the same computer as MedRec, you can enter http://localhost:7001/console.

  2. Enter your username and password.

    The administrator username and password are the ones you entered when creating the WebLogic Server samples domain.

  3. Then click Log In.

Building the MedRec Application Suite

Oracle provides the complete source code for the MedRec application. You can examine the source code to understand how individual features are implemented. You can also use the source code to understand how to organize and build Java EE applications using Ant and Maven.

Set Your Environment

  1. Open a command window.
  2. Set your environment by running either one of these scripts:
    • @MW_HOME/user_projects/domains/medrec/bin/setDomainEnv
    • @MW_HOME/wlserver/samples/server/setExamplesENV

You can now use Ant or Maven to rebuild MedRec from source and deploy it to the running MedRec Server.

Ant

  1. Change to the @MEDREC_HOME directory.
  2. Enter the following command:

     ant
  3. To disable running automated tests in the build process, use the -Dnotest=true flag:

     ant -Dnotest=true

Maven

  1. Change to the @MEDREC_HOME/modules directory.
  2. If you want a clean output directory, enter the following command:

     mvn clean
  3. To rebuild the MedRec application, enter the following command:

     mvn install

For more information about the build mechanism and project layout, refer to the specific chapters.

Troubleshooting

  • The server starts with the error java.net.BindException - Address already in use.

    This error means that the configured WebLogic Server port is being used by another process such as another WebLogic Server instance. To fix this, either reconfigure the port to a unique port or terminate the running process that is bound to the configured port.

  • You receive a Page cannot be displayed or an Error 404 - Not found error when attempting to access the server with a browser.

    This can happen if your browser is set up to access all URLs through a proxy server. If the machine running WebLogic Server is not registered with the proxy server, you will receive a URL failure from your browser. To fix this, you can configure your browser to not use the proxy server, or not use the proxy server to access an address that begins with the name of the machine running WebLogic Server.
