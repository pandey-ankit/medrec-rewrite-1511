The command-line build process is based on Apache Ant. The build process is responsible for compiling, testing, and deploying each module (sub-project) of MedRec.

Highlighted Features

In MedRec, each module is a Maven project and has its own pom file.

Common Commands

These are the most common default lifecycle phases executed:

  • validate: validates that the project is correct and that all necessary information is available
  • compile: compiles the source code of the project
  • test: tests the compiled source code using a suitable unit testing framework. These tests should not require that the code be packaged or deployed
  • package: takes the compiled code and packages it in its distributable format, such as a JAR file
  • install: installs the package into the local repository for use as a dependency in other projects locally

There is an additional Maven lifecycle phase of note beyond the default list above:

  • clean: cleans up artifacts created by prior builds

WebLogic Development Maven Plug-in

Apache Maven is a software tool for building and managing Java-based projects. WebLogic Server provides support for Maven through the provisioning of plug-ins that enable you to perform various operations on WebLogic Server from within a Maven environment.

The weblogic-maven-plugin provides enhanced functionality to install, start and stop server instances, create domains, execute WLST scripts, and compile and deploy applications. With the weblogic-maven-plugin, you can install WebLogic Server from within your Maven environment to fulfill the local WebLogic Server requirement.

For detailed information on installing and using the Weblogic Development Maven Plug-in to build applications and projects, see Developing Applications for Oracle WebLogic Server.

If you didn't install MedRec by Quick Start Configuration Wizard(choosing full Weblogic installation), please update Weblogic properties in @MEDREC_HOME/modules/pom.xml

Usage

To build and deploy the entire MedRec application, complete the following steps:

  1. Change to the @MEDREC_HOME/module directory, where MEDREC_HOME represents the directory in which the MedRec is configured. The default path is Oracle_Home/wlserver/samples/server/medrec.
  2. If you want a clean output directory, enter the following command:
    3.  mvn clean
  3. To rebuild the MedRec application, enter the following command:
    4.  mvn install
  4. To deploy the MedRec application on domain level, enter the following command:

    Deploy medrec.ear. Change to the @MEDREC_HOME/modules/medrec/medrec-assembly directory,

     mvn com.oracle.weblogic:weblogic-maven-plugin:deploy

    Deploy physician.ear. Change to the @MEDREC_HOME/modules/physician/physician-assembly directory,

     mvn com.oracle.weblogic:weblogic-maven-plugin:deploy

    Deploy chat.war. Change to the @MEDREC_HOME/modules/chat directory,

     mvn com.oracle.weblogic:weblogic-maven-plugin:deploy

    Note: due to different target types of each samples, deploy command only apply to Non-MT single server MedRec
