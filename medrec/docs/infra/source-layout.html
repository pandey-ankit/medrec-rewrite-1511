Module Hierarchy

The source code of the MedRec application is organized into a hierarchy of modules, under the /modules directory, as mentioned in the Introduction chapter. The hierarchy consists of two levels of sub-directories, dividing MedRec into various loosely-coupled sub-projects.

The first level of sub-directories divides the MedRec application into several modules with different business functionalities, such as the medrec EAR, physician EAR, common components, and so on.

The second level of sub-directories again divides the aforementioned medrec and physician modules into sub-modules that contain different technical building blocks following the layered architecture described in Design Overview. In general, these sub-directories are as follows:

  • /domain: Contains the domain models, packaged as an EJB JAR file.
  • /facade: Contains the facades, packaged as an EJB JAR file.
  • /xxx-facade-broker: Contains facade brokers of different types (such as JAX-WS and JAX-RS), each packaged as a Web application (WAR) file. Web services are typically contained in their own separate WAR.
  • /web: Contains web user interfaces, packaged as a WAR file.
  • /assembly: Contains the EAR's deployment descriptors, libraries, etc, used to assemble the preceding components into a complete EAR application.

Project Layout

Each leaf module in the hierarchy tree described above is a separate sub-project, following the project layout described by the Eclipse Web tools platform (WTP).
