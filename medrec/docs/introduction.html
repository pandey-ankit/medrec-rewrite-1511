What is MedRec?

Avitek Medical Records (or MedRec) is a WebLogic Server sample application suite that demonstrates all aspects of the Java Platform, Enterprise Edition (Java EE). MedRec is designed as an educational tool for all levels of Java EE developers. It showcases the use of each Java EE component, and illustrates best-practice design patterns for component interaction and client development. MedRec also illustrates best practices for developing and deploying applications with WebLogic Server on .

MedRec Application Overview

MedRec provides a framework for patients, doctors, and administrators to manage patient data. Patient data includes:

  • Patient profile information: A patient's name, address, social security number, and login information.
  • Patient medical record: Details about a patient's visit with a physician, such as the patient's vital signs and symptoms as well as the physician's diagnosis and prescriptions.

The MedRec suite consists of 3 main Java EE application modules, each packaged as an EAR or a WAR:

  • MedRec EAR: Patients log in this application to edit their profiles information, or request that their profiles are added to the system. Patients can also view prior medical records of visits with their physician. Administrators use this application to approve or deny new patient profile requests. The medrec EAR also provides all of the business logic used by the MedRec application suite, as well as the Web Services used by different clients.
  • Physician EAR: Physicians and nurses log in this application to search and access patient profiles, create and review patient medical records, and prescribe medicine to patients. The physician application is designed to communicate using the Web Services provided in medrec EAR.
  • Chat WAR: Physicians and patients chat with each other. One online physician, one opened chat room. There are two Websocket End Points, one for Patient, the other for Physician, and no web pages in this WAR. The medrec EAR and physician EAR contain chat room client connecting Websocket End Points with JavaScript.

Multitenant MedRec Domains

Note: WebLogic Server Multitenant domain partitions are deprecated in WebLogic Server 12.2.1.4.0 and will be removed in the next release. Oracle recommends that customers using domain partitions as a container dedicated to specific applications and resources consider the use of alternative container-based architectures, including the deployment of WebLogic applications and services in Docker containers running in Kubernetes clusters. Multitenant MedRec domains are deprecated in WebLogic Server 12.2.1.4.0 and will be removed in the next release.

In addition to the default single server domain, MedRec provides multi-tenant SaaS samples that run on three partitions, either on a single server or on a Coherence cluster. WebLogic Server Multi-Tenancy (WLS MT) provides resource isolation within a domain partition, which is an administrative and runtime slice of a WebLogic domain that is dedicated to running application instances and related resources for a tenant. Domain partitions achieve greater density by allowing application instances and related resources to share the domain, WebLogic Server itself, the Java virtual machine, and the operating system while isolating tenant-specific application data, configuration, and runtime traffic.

For more information, refer to the Multi-Tenancy Overview.

Project Directory

MedRec project files are installed into the MEDREC_HOME directory (MEDREC_HOME represents the directory in which the MedRec is configured).

The default path is @MW_HOME/wlserver/samples/server/medrec. The project directory contains the following subdirectory structure:

  • \dist: Deployable archives (JAR, WAR and EAR) generated by the modules of the MedRec application.
  • \docs: Documentation about the MedRec application suite.
  • \install: Configures MedRec using WLST.
  • \lib: Dependent libraries that may not be automatically available from WebLogic Server's general CLASSPATH, set in the @WL_HOME/server/bin/setWLSEnv command.
  • \modules: Source files of the different modules
    • \common: Common components used by the other modules
    • \medrec: Aforementioned MedRec EAR
    • \physician: Aforementioned physician EAR
    • \chat: Aforementioned chat WAR
    • \gar: Coherence Cache GAR
    • \medrec-data-import: Database initialization utility
    • \browser-starter: Helper EAR that automatically brings up a browser window for users after server starts up.
